Error Handling

POP3 Connection Failure
Returns DownloadResult(success=False) — pipeline continues with partial files already downloaded.
Missing Source File
Source-level FileNotFoundError → skip & continue. Only raises RuntimeError if all sources are missing.
Email Send Failure
Missing credentials or SMTP error → logged with full traceback, returns False. No crash.
Bad Email Date Header
Unparseable Date header → silently skipped. Scan continues to the next message.
POP3 Quit Error
Close failure → warning logged. Connection cleared in finally — no resource leak.
Partial Download
Returns success=False with partial received_times — failure alert accurately lists what arrived and what didn't.

Key Technical Highlights

Early Termination
Stops scanning the moment all files are found, or when it hits emails older than today.
Fault-Tolerant Consolidation
Missing source → skipped. Pipeline consolidates whatever arrived and reports what didn't.
Template-Driven Output
Data injected into a pre-built template — colors, freeze panes, filters, and pivot structure preserved every run.
2nd-Run Detection
Files saved with HHMM timestamp. Re-run checks for new files (≥ 19:00) → Updated report or No-Update notice.
Pivot Table in Email
Pivot data captured after refresh and embedded as HTML table in the email body.
Modular Pivot Utils
pivot_utils.py — pivot refresh (xlwings) + HTML rendering in one place.
